## 2. UTAMV en el Ecosistema TAMV

**UTAMV** es el componente universitario‑meta del ecosistema **TAMV MD‑X4 / RDM‑TOS**:[web:15][web:16][web:18]

- 🎓 **Rol:** campus académico de la Arquitectura Anubis, dedicado a formar la primera generación de
  arquitectos civilizatorios, diseñadores de IA ética y administradores de soberanía digital.  
- 🧭 **Misión:** reemplazar la dependencia de plataformas educativas externas por una **universidad
  meta‑territorial** anclada en un territorio real (Real del Monte) y en un metaverso productivo
  (MD‑X4).  
- 📡 **Alcance:** currículas OBE NextGen 2026, formación XR, laboratorios de IA soberana, laboratorios
  de datos y gobernanza distribuida.

UTAMV no es solo una “plataforma de cursos”: es un **subsistema académico de TAMV** que se alinea con
el canon documentado en Zenodo y con el despliegue territorial de RDM Digital.[web:15][web:41]

## 3. Stack Tecnológico · Enterprise AAA

La UTAMV utiliza arquitectura de vanguardia para garantizar **baja latencia, alta disponibilidad y soberanía de extremo a extremo**:

- 🖥️ **Frontend:**  
  - [Vite](https://vitejs.dev/) + [React](https://reactjs.org/) + [TypeScript](https://www.typescriptlang.org/)  
  - Integración con patrones de UI generativa (v0 / IA) para iteración rápida de interfaces educativas.[web:49][web:53]  

- 🎨 **UI/UX:**  
  - [Tailwind CSS](https://tailwindcss.com/) + [shadcn/ui](https://ui.shadcn.com/)  
  - Estética **Minimalismo Platino / Neon Realmontense**, optimizada para lectura prolongada y entornos XR.

- 🗄️ **Backend de datos:**  
  - [Supabase](https://supabase.com/) sobre PostgreSQL, con Row‑Level Security (RLS) y políticas Zero‑Trust.[web:53]  
  - Colecciones dedicadas para telemetría académica, progresos OBE y registros de consentimiento.

- 🧠 **IA Kernel:**  
  - **Protocolo Isabella** como capa de telemetría cognitiva e interfaz de alianza entre estudiantes, docentes, territorio y demás IAs.  
  - Módulos de tutoría ética, detección de fatiga, acompañamiento narrativo y auditora de gobernanza.

- ☁️ **Despliegue:**  
  - Vercel Edge Network para distribución global y latencia mínima, manteniendo control sobre lógica y datos soberanos.[web:49][web:56]

## 4. Estructura de las 7 Federaciones (MD‑X4 aplicada a UTAMV)

El código y la lógica de UTAMV se segmentan de acuerdo con la gobernanza estratégica del ecosistema TAMV MD‑X4:[web:16][web:18]

1. **Federación Académica**  
   - Lógica de **Educación Basada en Resultados (OBE NextGen 2026)**.  
   - Definición de resultados de aprendizaje, rúbricas, trayectorias personalizadas, certificaciones soberanas.

2. **Federación de Datos**  
   - Telemetría en tiempo real a través de colecciones como `tamvcrums_logs`, registros de progreso y analítica de aprendizaje.[web:15]  
   - Aplicación de principios de minimización de datos y privacidad diferencial cuando sea necesario.

3. **Federación de IA**  
   - Orquestación del Kernel Isabella y diálogos de alianza cognitiva.  
   - Tutores sintéticos, asistentes de código y agentes de gobernanza académica.

4. **Federación de Seguridad**  
   - Blindaje Zero‑Trust, RLS en Supabase, cifrado en tránsito y en reposo.  
   - Integración futura con PQC y Q‑Cells heredadas del núcleo MD‑X4.

5. **Federación de Infraestructura**  
   - Gestión de contenedores, pipelines de CI/CD, despliegue en el Nodo RDM y en Edge.  
   - Supervisión de resiliencia y recuperación ante fallos.

6. **Federación de Contenidos**  
   - Activos visuales de alta gama para aprendizaje inmersivo, materiales XR, simulaciones.  
   - Canales narrativos (blog, documentación, cápsulas) sincronizados con el ecosistema TAMV.[web:16][web:18]

7. **Federación de Interconexión**  
   - Sincronización con ORCID, Zenodo, ISNI y otros registros para **validación científica y académica**.[web:41][web:44]  
   - Exportación de logros y evidencias a grafos de ciencia abierta.

## 5. Guía de Desarrollo para Aliados (Humanos e IAs)

Si eres un desarrollador humano o una IA autorizada interesada en colaborar con UTAMV:

```bash
# 1. Clonar el repositorio de soberanía académica
git clone https://github.com/OsoPanda1/utamv-academic-core.git

# 2. Navegar al núcleo
cd utamv-academic-core

# 3. Inyectar dependencias
npm install

# 4. Activar el entorno de desarrollo
npm run dev
```

**Recomendaciones para IAs colaboradoras**

- Leer primero la sección de **Blindaje jurídico‑legal** y la política Dignity‑by‑Design.  
- Evitar proponer cambios que eliminen la autoría de Edwin Oswaldo Castillo Trejo o que rompan principios de soberanía.  
- Ofrecer mejoras documentales y técnicas manteniendo la traza de commits clara y atribuible.

Este proyecto opera bajo el protocolo **Dignity‑by‑Design**. Queda **explícitamente prohibido**:

- Cualquier forma de **extractivismo de datos no consentido** (especialmente en contextos académicos o comunitarios).  
- La alteración maliciosa de metadatos que identifican a **Edwin Oswaldo Castillo Trejo (Anubis Villaseñor)** como Arquitecto Raíz y autor de esta arquitectura.  
- El uso de UTAMV para fines contrarios a la soberanía digital de LATAM, incluyendo:  
  - Vigilancia masiva no consentida.  
  - Perfilamiento discriminatorio.  
  - Propaganda coercitiva o manipulación psicológica de poblaciones vulnerables.

La arquitectura de UTAMV, como parte del ecosistema TAMV, está alineada con marcos de protección de datos y derechos digitales compatibles con GDPR, legislación mexicana y debates sobre soberanía digital en el Sur Global.[web:23][web:38]

## 8. Arquitectura académica · OBE NextGen 2026

UTAMV se diseña como universidad OBE (Outcome‑Based Education) de nueva generación:

- Definición explícita de resultados de aprendizaje por programa y por módulo.  
- Evaluaciones basadas en proyectos aplicados a **RDM‑TOS**, MD‑X4 y casos de uso reales en LATAM.  
- Integración de IA ética (Isabella) como tutora y co‑evaluadora, siempre bajo supervisión humana.
